    \section*{1 Problema}
      \textbf{Diseña un juego y explica por qué es un juego. Además, da una estrategia 
      ganadora y las reglas}

      Intentaremos explicar el juego en forma normal, este sera de dos jugadores.
      Supongamos un natural arbitrario $n$,
      Entonces el juego:
      $G =(S_0, S_1; u_0, u_1)$

      Estos representan:
      \begin{itemize}
        \item $S_i$: El conjunto de las estrategias posibles para el jugador $i$
        \item $u_i$: La funcion de recompensa dadas las estrategias
      \end{itemize}

      Pero bueno, intentamos explicar las reglas del juego en palabras sencillas, 
      tenemos a estos dos jugadores
      y el tablero que inicialmente tendra $n$ elementos, digamos piedras para hacer la cosa
      mas sencilla de explicar. Cada jugador puede tomar de la pila la cantidad de piedras que quiera
      excepto:
      \begin{itemize}
        \item En el primer movimiento donde no se pueden tomar todas las piedras
        \item En movimientos posteriores no se puede retirar la $q$ piedras si es que antes
        alguna otra jugada habia sido retirar $q$ piedras
      \end{itemize}.

      El primer jugador que se quede sin movimientos validos que hacer pierde.

      Por ejemplo veamos varios escenarios:
      Tenemos 3 piedras.
      \begin{itemize}
        \item Alicia puede tomar 2 piedras
        \item Beto puede tomar 1 piedra, la toma
        \item Alicia no puede hacer nada, gana Beto
      \end{itemize}

      Tenemos 4 piedras
      \begin{itemize}
        \item Alicia puede tomar 2 piedras
        \item Beto no puede tomar 2 piedras, asi que toma 1
        \item Alicia no puede tomar 1, gana Beto
      \end{itemize}

      Ahora veamos una estrategia ganadora:
      \begin{itemize}
        \item Si es tu turno, la unica forma de evitar que pierdas en el siguiente turno es
        evitar que el oponente se lleve todas las piedras disponibles en el tablero y eso
        solo se puede hacer si es que quedan una cantidad par de piedras, tomas la mitad
        de las mismas como tu jugada y entonces el oponente ya no tiene una jugada que te haria perder
        inmediatamente
        \item Por lo mismo podemos deducir que si tenemos enfrente una cantidad impar de piedras
        ya no hay estrategia ganadora
      \end{itemize}

      Esta idea la podemos codificar por ejemplo en un programa (haciendo un poco mas eficiente la idea
      de encontrar cuantas veces un numero de piedras se puede dividir entre dos):

      \begin{lstlisting}[language=C, gobble=4]
        inline auto is_even(const int x) -> bool { return x & 1 == 0; }
        inline auto is_odd(const int x) -> bool { return x & 1 == 1; }
        
        inline auto doesPlayerOneWin(const int num_balls) -> bool {
          if (is_odd(num_balls)) return false;
          
          const auto num_of_zeros_at_beginning = static_cast<int>(log2(num_balls & -num_balls));
          return is_odd(num_of_zeros_at_beginning);
        }
        \end{lstlisting}

    \section*{2 Problema}
      \textbf{Da un ejemplo (dibuja) de una digráfica de orden 8 que...}

      \begin{itemize}
        \item fuertemente conexa

          Sencillo, una grafica fuertemente conexa es aquella en la que dados cuales quiera 
          dos vertices diferentes hay una forma de llegar de uno a otro

          \begin{figure}[h]
              \includegraphics[width=0.3\textwidth]{a}
          \end{figure}


        \item unilateralmente conexa que no sea fuerte

          Esta esta sencillo, la idea es que dados dos vertices $u, v$ haya una forma de llegar
          o de $u$ a $v$ o de $v$ a $u$.
          Pero tiene que haber al menos un vertices para el que o no se pueda llegar de 
          $u$ a $v$ o que no se pueda llegar de $v$ a $u$.

          \begin{figure}[h]
            \includegraphics[width=0.3\textwidth]{b}
          \end{figure}

          Por ejemplo aqui no podemos llegar del nodo azul al naranja.

        \item fuerte con pozos
        
          Esta esta interesante, osea, queremos una digrafica fuertemente conexa pero que al mismo tiempo
          al menos dos de sus vertices sean pozos.
          Tomemos esos dos vertices, como suponemos que es fuertemente conexa tiene que haber una forma
          de llegar de uno al otro y viceversa, pero es que son pozos por lo que el camino no puede empezar
          en ninguna de ellas, por lo tanto es imposible que exista una grafica que sea fuerte y con pozos
          a la vez.

      \end{itemize}
